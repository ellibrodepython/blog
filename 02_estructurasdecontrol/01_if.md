# Condicional en Python

De no ser por las **estructuras de control**, el c√≥digo en cualquier lenguaje de programaci√≥n ser√≠a ejecutado secuencialmente hasta terminar. Un c√≥digo, no deja de ser un conjunto de instrucciones que son ejecutadas unas tras otra. Gracias a las estructuras de control, podemos **cambiar el flujo de ejecuci√≥n de un programa**, haciendo que ciertos bloques de c√≥digo se ejecuten si y solo si se dan unas condiciones particulares.

## Uso del if

Un ejemplo ser√≠a si tenemos dos valores `a` y `b` que queremos dividir. Antes de entrar en el bloque de c√≥digo que divide `a/b`, ser√≠a importante verificar que `b` es distinto de cero, ya que la divisi√≥n por cero no est√° definida. Es aqu√≠ donde entran los condicionales `if`.


```python
a = 4
b = 2
if b != 0:
    print(a/b)
```


En este ejemplo podemos ver como se puede usar un `if` en Python. Con el operador `!=` se comprueba que el n√∫mero `b` sea distinto de cero, y si lo es, se ejecuta el c√≥digo que est√° indentado. Por lo tanto un `if` tiene dos partes:
* La **condici√≥n** que se tiene que cumplir para que el bloque de c√≥digo se ejecute, en nuestro caso `b!=0`.
* El **bloque de c√≥digo** que se ejecutar√° si se cumple la condici√≥n anterior.

Es muy importante tener en cuenta que la sentencia `if` debe ir terminada por `:` y el bloque de c√≥digo a ejecutar debe estar indentado. Si usas alg√∫n editor de c√≥digo, seguramente la indentaci√≥n se producir√° autom√°ticamente al presionar enter. N√≥tese que el bloque de c√≥digo puede tambi√©n contener m√°s de una l√≠nea, es decir puede contener m√°s de una instrucci√≥n.


```python
if b != 0:
    c = a/b
    d = c + 1
    print(d)
```



Todo lo que vaya despu√©s del `if` y est√© indentado, ser√° parte del bloque de c√≥digo que se ejecutar√° si la condici√≥n se cumple. Por lo tanto el segundo `print()` "Fuera if" ser√° ejecutado siempre, ya que est√° fuera del bloque `if`.


```python
if b != 0:
    c = a/b
    print("Dentro if")
print("Fuera if")
```


Existen otros operadores que se ver√°n en otros cap√≠tulos, como el de comparar si un n√∫mero es mayor que otro. Su uso es igual que el anterior.


```python
if b > 0:
    print(a/b)
```



Se puede tambi√©n combinar varias condiciones entre el `if` y los `:`. Por ejemplo, se puede requerir que un n√∫mero sea mayor que 5 y adem√°s menor que 15. Tenemos en realidad tres operadores usados conjuntamente, que ser√°n evaluados por separado hasta devolver el resultado final, que ser√° `True` si la condici√≥n se cumple o `False` de lo contrario.


```python
a = 10
if a > 5 and a < 15:
    print("Mayor que 5 y menos que 15")
```

Es muy importante tener en cuenta que a diferencia de en otros lenguajes, en Python no puede haber un bloque `if` vac√≠o. El siguiente c√≥digo dar√≠a un `SyntaxError`.


```python
if a > 5:
```

Por lo tanto si tenemos un `if` sin contenido, tal vez porque sea una tarea pendiente que estamos dejando para implementar en un futuro, es necesario hacer uso de `pass` para evitar el error. Realmente `pass` no hace nada, simplemente es para tener contento al interprete de c√≥digo.


```python
if a > 5:
    pass
```

Algo no demasiado recomendable pero que es posible, es poner todo el bloque que va dentro del `if` en la misma l√≠nea, justo a continuaci√≥n de los `:`. Si el bloque de c√≥digo no es muy largo, puede ser √∫til para ahorrarse alguna l√≠nea de c√≥digo.


```python
if a > 5: print("Es > 5")
```


Si tu bloque de c√≥digo tiene m√°s de una l√≠nea, se pueden poner tambi√©n en la misma l√≠nea separ√°ndolas con `;`.


```python
if a > 5: print("Es > 5"); print("Dentro del if")
```



## Uso de else y elif

Es posible que no solo queramos hacer algo si una determinada condici√≥n se cumple, sino que adem√°s queramos hacer algo de lo contrario. Es aqu√≠ donde entra la cl√°usula `else`. La parte del `if` se comporta de la manera que ya hemos explicado, con la diferencia que si esa condici√≥n no se cumple, se ejecutar√° el c√≥digo presente dentro del `else`. N√≥tese que ambos bloque de c√≥digo son excluyentes, se entra o en uno o en otro, pero nunca se ejecutar√°n los dos.


```python
x = 5
if x == 5:
    print("Es 5")
else:
    print("No es 5")
```


Hasta ahora hemos visto como ejecutar un bloque de c√≥digo si se cumple una instrucci√≥n, u otro si no se cumple, pero no es suficiente. En muchos casos, podemos tener varias condiciones diferentes y para cada una queremos un c√≥digo distinto. Es aqu√≠ donde entra en juego el `elif`.


```python
x = 5
if x == 5:
    print("Es 5")
elif x == 6:
    print("Es 6")
elif x == 7:
    print("Es 7")
```


Con la cl√°usula `elif` podemos ejecutar tantos bloques de c√≥digo distintos como queramos seg√∫n la condici√≥n. Traducido al lenguaje natural, ser√≠a algo as√≠ como decir: si es igual a 5 haz esto, si es igual a 6 haz lo otro, si es igual a 7 haz lo otro.

Se puede usar tambi√©n de manera conjunta todo, el `if` con el `elif` y un `else` al final. Es muy importante notar que `if` y `else` solamente puede haber uno, mientras que `elif` puede haber varios.


```python
x = 5
if x == 5:
    print("Es 5")
elif x == 6:
    print("Es 6")
elif x == 7:
    print("Es 7")
else:
    print("Es otro")
```



Si vienes de otros lenguajes de programaci√≥n, sabr√°s que el `switch` es una forma alternativa de `elif`, sin embargo en Python esta cl√°usula no existe.

## Operador ternario

El operador ternario o `ternary operator` es una herramienta muy potente que muchos lenguajes de programaci√≥n tienen. En Python es un poco distinto a lo que ser√≠a en C, pero el concepto es el mismo. Se trata de una cl√°usula `if`, `else` que se define en una sola l√≠nea y puede ser usado por ejemplo, dentro de un `print()`.

<p class="alert alert-info"> 
<b>Para saber m√°s:</b> El operador ternario fue propuesto <a href="https://www.python.org/dev/peps/pep-0308/"> en la PEP 308.</a>
</p>


```python
x = 5
print("Es 5" if x == 5 else "No es 5")
#Es 5
```


Existen tres partes en un operador ternario, que son exactamente iguales a los que hab√≠a en un `if` `else`. Tenemos la condici√≥n a evaluar, el c√≥digo que se ejecuta si se cumple, y el c√≥digo que se ejecuta si no se cumple. En este caso, tenemos los tres en la misma l√≠nea.


```python
# [c√≥digo si se cumple] if [condici√≥n] else [c√≥digo si no se cumple]
```

Es muy √∫til y permite ahorrarse algunas l√≠neas de c√≥digo, adem√°s de aumentar la rapidez a la que escribimos. Si por ejemplo tenemos una variable a la que queremos asignar un valor en funci√≥n de una condici√≥n, se puede hacer de la siguiente manera. Siguiendo el ejemplo anterior, en el siguiente c√≥digo intentamos dividir `a` entre `b`. Si `b` es diferente a cero, se realiza la divisi√≥n y se almacena en `c`, de lo contrario se almacena `-1`. Ese `-1` podr√≠a ser una forma de indicar que ha habido un error con la divisi√≥n.


```python
a = 10
b = 5
c = a/b if b!=0 else -1
print(c)
#2
```
